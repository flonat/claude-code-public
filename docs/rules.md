# Rules

> 9 auto-loaded policies that Claude follows in every session.

Rules are markdown files in `.claude/rules/` that define always-on policies. Unlike skills (invoked on demand) or agents (launched as sub-processes), rules are loaded into every session automatically and shape Claude's behaviour at all times.

## Available Rules

| Rule | Category | Purpose |
|------|----------|---------|
| `plan-first.md` | Workflow | Draft and save a plan before multi-file changes |
| `scope-discipline.md` | Workflow | Only make changes explicitly requested |
| `learn-tags.md` | Knowledge | Record corrections as `[LEARN:category]` tags |
| `read-docs-first.md` | Workflow | Check project docs before searching |
| `lean-claude-md.md` | Documentation | Keep CLAUDE.md files lean — pointer pattern |
| `overleaf-separation.md` | Project Structure | No code or data in `paper/` directories |
| `ignore-agents-md.md` | Safety | Never read `AGENTS.md` files (external agent output) |
| `ignore-gemini-md.md` | Safety | Never read `GEMINI.md` files (external agent output) |
| `design-before-results.md` | Research Integrity | Lock research design before examining point estimates |

## Rule Details

### plan-first.md

**When it applies:** Multi-file edits (3+ files), new features, unclear scope, refactoring.

**Protocol:**
1. Draft a plan before writing any code
2. Save the plan to `log/plans/YYYY-MM-DD_description.md`
3. Get approval before implementing
4. Implement after approval

**Quick mode:** Skip planning when the user says "quick", "try this", "experiment", or "skip planning".

### scope-discipline.md

**Principle:** Only make changes the user explicitly requested. Don't fix additional issues unless asked.

If you notice something else: complete the requested task first, then mention what you spotted and ask permission.

### learn-tags.md

**Format:** `[LEARN:category] Incorrect → Correct`

**Categories:** `notation`, `citation`, `code`, `method`, `domain`

Records corrections to `MEMORY.md` immediately when discovered. Also populates structured knowledge tables (Notation Registry, Citations, Anti-Patterns, Code Pitfalls).

### read-docs-first.md

**Principle:** Never explore when documentation already answers your question.

**On every session:** Read context files in order before doing any work:
1. `CLAUDE.md` (auto-loaded)
2. `.context/current-focus.md`
3. `.context/projects/_index.md`
4. `MEMORY.md`
5. Latest session log
6. Latest plan

### lean-claude-md.md

**Principle:** CLAUDE.md is loaded every session — every line costs tokens. Use the pointer pattern: one-line summary + link to detailed docs.

**Thresholds:** CLAUDE.md > 200 lines → review for extractable content. Any section > 15 lines of reference material → extract to `docs/`.

### overleaf-separation.md

**Principle:** The `paper/` directory is for LaTeX source files ONLY. Code, data, and computational artifacts belong in the project directory — never inside `paper/`.

**Allowed in `paper/`:** `.tex`, `.sty`, `.cls`, `.bst`, `.bbl`, `.bib`, figures (`.pdf`, `.png`, `.eps`), `.latexmkrc`, `out/`.

**Never in `paper/`:** `.py`, `.R`, `.csv`, `.xlsx`, `.ipynb`, `.venv/`, `requirements.txt`.

### ignore-agents-md.md / ignore-gemini-md.md

Skip files named `AGENTS.md` or `GEMINI.md` — these are generated by other AI tools and are not part of Claude's context.

### design-before-results.md

**Principle:** Lock the research design before examining point estimates. Specify the estimand, identification strategy, and analysis plan before looking at results.

**The falsifiability test:** Before running any analysis, ask: "If I specified my analysis plan before running it, would I change anything about what I'm about to do?"

## Global Access

Rules are globally available via a symlink created by `setup.sh`:

```
~/.claude/rules/ → <repo>/.claude/rules/
```

## Creating New Rules

1. Create a markdown file in `.claude/rules/`
2. Write the policy clearly with "When this applies" and "What to do" sections
3. The rule is automatically loaded in every session
